Roles & Permissions

Understand the 6-tier role-based access control system and what each role can do.

1 RBAC Overview

Multi-Project Tracker uses a 6-tier Role-Based Access Control (RBAC) system:

Tier 1: System Administrator (highest)
Tier 2: Project Admin
Tier 3: Project Manager
Tier 4: Team Member
Tier 5: Stakeholder
Tier 6: External Viewer (lowest)

Each role inherits permissions from lower tiers and adds additional capabilities.

1 System Administrator

Full system access - highest privilege level

Permissions:

  • ✓ Create, manage, and delete all projects system-wide
  • ✓ Manage all users and their system roles
  • ✓ Access and modify any project regardless of membership
  • ✓ Configure system settings and integrations
  • ✓ All capabilities of lower-tier roles

Assignment: Set during user account creation by existing System Administrators

2 Project Admin

Full control over specific projects

Permissions:

  • ✓ Delete the project
  • ✓ Modify project settings and configuration
  • ✓ Manage team members and their roles
  • ✓ Create, edit, and delete all issues and action items
  • ✓ Generate all reports
  • ✓ Full comment and mention privileges

Assignment: Set by System Administrators or during project creation

3 Project Manager

Team and content management capabilities

Permissions:

  • ✓ Invite new team members
  • ✓ Manage existing team members (except admins)
  • ✓ Create, edit, and delete all issues and action items
  • ✓ Generate reports
  • ✓ Moderate comments (delete any comment)
  • ✗ Cannot delete the project
  • ✗ Cannot modify project core settings

Assignment: Set by Admins or Managers when inviting team members

4 Team Member

Regular contributor with editing rights

Permissions:

  • ✓ Create new issues and action items
  • ✓ Edit own issues and action items
  • ✓ Update status on assigned items
  • ✓ Comment on all items
  • ✓ Use @mentions
  • ✓ View all project data
  • ✗ Cannot delete items created by others
  • ✗ Cannot manage team
  • ✗ Cannot delete comments by others

Assignment: Default role when invited to a project

6 Viewer (Stakeholder / External)

Read-only access for oversight

Permissions:

  • ✓ View all issues and action items
  • ✓ View all comments
  • ✓ Access dashboard and reports
  • ✓ View team members
  • ✗ Cannot create or edit any content
  • ✗ Cannot comment
  • ✗ Cannot use @mentions

Assignment: Set by Managers or Admins for stakeholders and external observers
