Admin Tools

System Administrator tools for managing data quality and system maintenance.

1 Admin Tools Overview

Admin Tools provide System Administrators with utilities to:

  • Maintain Data Quality: Fix data inconsistencies and errors
  • Correct Assignee Names: Update mismatched assignee names to match actual usernames
  • Improve Report Accuracy: Ensure Team Performance Reports show correct data
  • Bulk Operations: Update multiple items efficiently

⚠️ System Administrator Only:

Admin Tools are only accessible to users with the System Administrator role. These tools can modify data across projects, so use them carefully.

2 Accessing Admin Tools

To access Admin Tools:

  1. Log in with a System Administrator account
  2. Click the hamburger menu (three lines) in the top navigation
  3. Look for the "Admin Tools" option in the menu
  4. Click "Admin Tools" to access the admin dashboard

Don't See Admin Tools?

The Admin Tools link only appears for System Administrators. If you need admin access, contact your system administrator to upgrade your role.

3 Assignee Name Correction Tool

What It Does

This tool fixes mismatched assignee names that can cause Team Performance Reports to show incorrect or missing data. When items are assigned manually with names that don't exactly match usernames in the system (e.g., "Sri Hari" vs "Srihari S"), reports fail to match the assignments correctly.

Common Problems It Solves

  • Team reports showing incomplete assignment data
  • Assignee names with typos or variations (e.g., "Sakthi S4" instead of "Sakthi")
  • Manually entered names that don't match exact usernames
  • Missing data in completion rate calculations

How It Works:

The tool scans all issues and action items, identifies assignee names that don't match any valid username, and allows you to correct them in bulk.

4 Using the Assignee Correction Tool

Follow these steps to correct assignee names:

  1. Access the Tool: Navigate to Admin Tools and select "Assignee Name Correction"
  2. Review Mismatches: The tool displays all mismatched assignee names highlighted in yellow
  3. See Statistics: View how many issues and action items are affected by each mismatch
  4. Select Correct Username: For each mismatched name, choose the correct username from the dropdown
  5. Bulk Update: Click "Update All Assignees" to apply corrections
  6. Confirm Changes: Confirm the bulk update when prompted
  7. Verify Success: See a success message showing how many items were updated

Example:

Mismatch: "Sri Hari" (6 issues, 3 action items)
→ Select: "Srihari S" from dropdown
→ Update: All 9 items now assigned to "Srihari S"

After Correction

Once corrected:

  • Team Performance Reports will show accurate assignment data
  • Completion rates will be calculated correctly
  • Assignee names will match exactly with system usernames
  • Future assignments should use the username picker to avoid mismatches

5 Best Practices

✓ Run Periodic Checks

Check for mismatches monthly or after importing data from external sources

✓ Verify Before Updating

Double-check that you're selecting the correct username for each mismatch before bulk updating

✓ Use Assignment Pickers

Train team members to use the assignee dropdown when creating items instead of typing names manually

✓ Document Changes

Keep a record of bulk updates in case you need to reference what was changed

✗ Don't Rush Bulk Updates

Bulk updates affect multiple items at once. Review carefully before confirming to avoid assigning items to wrong users

💡 Pro Tip

After running the Assignee Correction Tool, regenerate any Team Performance Reports to see the corrected data. The reports will now accurately reflect workload distribution and completion rates.
